Http header （http 1.1）

2020-02-26

cache-Control

指定请求和响应遵守的缓冲机制。

  • max-age=<seconds> 缓存存储最大周期
  • no-cache 缓存副本之前，强制把缓存request提交给原server验证
  • no-store 不使用缓存
  • no-transform 不可对资源进行转换或转变。Content-*不可由proxy修改

Date

http报文发送时间，时间表示为世界标准时。

Pragma

用来实现特定指令，常用 Pragma:no-cache

Connection

当前事务处理完后是否关闭连接。(default: keep-alive)

Keep-Alive

允许消息发送者设置超时时长和最大请求数。Connection必须为 keep-alive时生效

Content-Disposition

指明回复内容的展示形式(内联inline:页面的一部分，附件attachment:下载并保存本地)

或者在request中表单POST请求用到一个值：form-data.

Content-Type

在实体头部用于指明资源的MIME类型

boundary包含1-70个字符，用于封装消息的多个部分边界

request header：

Content-Encoding

表明对特定媒体数据进行压缩，与Accept-Encoding对应

Content-Length

表明发送给接收方消息主体的大小(8位10进制)

Content-Location

返回实体数据的地址，提供可访问的资源的直接下载地址，返回实体。

Request

  • 请求行：表明请求类型及http版本 (post请求) （第一行）

    请求包含：GET, POST, HEAD, PUT, DELETE, TRACK

  • 请求头部：表明附加信息 （第二行开始）

  • 空行：必有 （倒数第二行）

  • 数据主体 （最后一行）

Cache-Control

  • max-stale=[=<seconds>] client可接收过期时间内的资源
  • min-fresh=<seconds>
  • only-if-cached client只接受已缓存的response，并不向server发送request检查是否更新

Host

指定请求资源的主机和端口号，请求url的原有服务器或网关的位置。

Accept

client可接收的类型

  • / 单一精确的MIME类型(eg: text/html)
  • /* 某一类MIME类型(eg: image/* = {image/png, imagegif, …})
  • */* 任意的MIME类型

Accept-Charset

client接收的字符集，如果server不能匹配则返回 406.

例如 utf-8, iso-8859-1

Accept-Encoding

client能理解的内容编码方式(一般为压缩算法)，server可以选择不压缩

参数包含：gzip, compress, deflate, br, identity, *

User-Agent

client信息(浏览器、操作系统信息)

Referer

包含当前请求页面的来源页面地址，表示当前页面是通过此来源的连接进入的。

以下情况不会发送Referer：

  • 来源页面采用的协议为表示本地文件或者数据的URI
  • 当前请求页面采用http协议，而来源页面采用https

Range

请求实体的一个或多个子范围(server可忽略)

Cookie

将先前通过Set-Cookie首部放入client的Cookie的值发送给server

Authorization

client的身份凭证，在server返回401或www-authenticate消息头之后后续发送的request包含此消息头

Content-Disposition

作为multipart body中的信息头，用于表单的POST请求，且只有name和filename参数可用于http中

If-Match / If-None-Match

对于GET和HEAD方法，用来匹配ETag的值，来判断是否更新资源

对于PUT方法，设值为*，用来保证上传不会覆盖原始资源，防止之前数据丢失。

Response

  • 状态行：http version 状态码 状态消息 （第一行）
    • 1xx: 信息响应，接收请求后继续处理
    • 2xx: 处理成功
    • 3xx: 重定向响应
    • 4xx: client错误
    • 5xx: server错误
  • 消息报头：附加信息 (第二行开始)
  • 空行： 必有
  • 响应正文

Expries

response过期时间

Cache-Control

  • public 表明response可被任何对象(client,proxy, etc.)缓存(共享缓存)
  • private 表明response只能被单个用户缓存(proxy不可缓存)(私有缓存)
  • must-revalidate 资源过期时，在成功向server验证之前，不能使用该资源响应后续request
  • proxy-revalidate 同must-revalidate，但只作用于共享缓存(proxy)
  • s-maxage=<seconds> 覆盖max-age或Expries header，作用于共享缓存

Accept-Patch

通知client哪些media-type能被server理解，如果request发送的media-type不能解析，则返回415

Age

ETag

资源特定版本标识符。当内容变化时ETag防止资源同时更新时相互覆盖。如果url中的资源更改则一定生成新的ETag值。

W/表示开启弱验证器

Location

需要将页面重定向到的地址，配合返回码3xx。其中303保留GET内容，307和308不转发原有方法。

Set-Cookie

用于response向client发送cookie

options包含：

  • Expries=<date> cookie最长有效时间
  • Max-Age=<non-zero-digit> 有效保持时间
  • Domain=<domain-value> cookie可送达的主机名，默认为当前访问地址中的主机
  • HttpOnly cookie不能使用js由Document.cookie、XMLHttpRequest,Request调用，防止XSS
  • Secure 带有安全属性的cookie，只允许在https中设置
  • Path=<path-value> 指定一个url路径，该路径必须出现在资源请求路径的中。

WWW-Authenticate

指定获取资源对接的验证方式，一般与401一起发送

realm 为一个保护区域的描述，可选。

Message Queuing

2020-02-18

AMQP(advance message queuing protocol)

面向消息中间件的应用层协议，关注于消息以字节流传递的方式，保证消息的排序和路由的可靠性和安全性。并不保证API的一致性。具体见https://www.amqp.org/

Advance Message Queuing Model

AMQ把消费者和生产者隔离，使得任何一方出现问题时不影响另一方的正常信息获取和发送。

  1. Message Queue

    消息队列(queue)将生存者(producer)发出的消息存储在内存或外存中，并按照一定顺序转发给一个或多个消费者(consumer)，每个队列是独立隔离的。

  2. Exchange

    交换机用于接收生产者发出的消息，然后基于路由(routing)和队列(queue)绑定关系(binding)进行匹配，投递给对应的交换机(exchange)或队列分发。交换机不存储信息，只进行分发。

Message flowing

  • producer创建消息同时设置消息属性并标记路由信息；
  • 当消息传递到exchange时，由exchange判断能否转发，如果不能则丢弃消息并反传回producer;
  • 一个exchange可以转发到多个queue，由此转发的queue内容相同，但操作隔离;
  • 当message到达queue时，根据AMQP投递给consumer,若没有consumer则将message存入内存或外存；
  • 当message传递给consumer时，将message从缓冲区(内存或外存)删除。

MQ使用场景

  1. 解耦：producer和consumer任何一方出现问题时都不影响另外一方
  2. 异步处理：当服务的其中几步可以异步操作时，可以通过MQ通知两个步骤同时操作
  3. 秒杀活动：将用户的购买信息写入消息队列，如果队列满则丢弃数据，然后由秒杀业务进行信息读取并处理
  4. 日志处理：对于大量日志传输问题，主要应用kafka进行。

Cycle-Consistent Deep Generative Hashing for Cross-Modal Retrieval

2019-04-19

from TIP 2019

abstract

由于生成的hash的对于输入的对象会有信息丢失，本文利用cycle-GAN，通过将生成的hash返回生成输入，从而减少在有模态对象生成hash code的过程中信息的丢失。同时利用随机hash生成结合概率模型获得loss函数。

introduction

由于很多实际因素，成对的训练数据在未来将难以实现，所以本文接用DGH(Deep Generative Hashing)将不同模态对象的提取的特征与hashing进行相互生成，从而减少因feature map到hashing之间的信息损失。同时结合cycle-consistency，将不同模态之间的对象特征进行相互转换，利用GAN的原理，增强分类器对于来自不同模态特征所生成的该模态下特征的分类能力。以图像与文本两个模态为例。

按照上述图中的描述过程，映射G表示由图像特征生成文本特征，F表示由文本特征生成图像特征。其中映射的过程包含由输入特征到hashing的映射H，以及由H映射出的hashing对输入特征的反映射P，最后再由P反映射出的输入特征来映射生成另外一个模态的特征。具体过程在上图中标出。

Network and training objective

network structure

本文涉及到3个过程：

  1. 不同模态对象的特征提取网络
  2. feature map与hash互相生成网络
  3. 不同模态feature map的互相生成与辨别网络

在文中明确说明了文本模态的特征提取网络为“Generative Adversarial Text to Image Synthesis”中的文本编码模型，从该参看文献中引用对于该模型的描述。

The description embedding ϕ(t) is first compressed using a fully-connected layer to a small dimension (in practice we used 128) followed by leaky-ReLU and then concatenated to the noise vector z

此外，文章在实验中采用了CNN-F，fc7的输出作为图像的提取特征。

loss and training objective

adversarial loss

adversarial loss为图像特征与文本特征互相生成的loss。G表示图像特征生成文本特征，F表示文本特征生成图像特征，xu表示图像特征，xv表示文本特征。Dxu表示用于区分生成的图像特征{F(xv)}与原生图片特征{xu}，DxV表示用于区分生成的文本特征{F(xu)}与原生文本特征{xV}.其中G和H的定义在introduction中有介绍，其中的H,P两个过程的loss将在后续定义。
$$
L_{GAN}(G,D_{x_v},x_u,x_v)=E_{x_v \thicksim p_{data}(x_v)}[log D_{x_v}(x_v)] + E_{x_u \thicksim p_{data}(x_u)}[log (1 - D_{x_v}(G(x_u)))]
$$

$$
L_{GAN}(G,D_{x_u},x_v,x_u)=E_{x_u \thicksim p_{data}(x_u)}[log D_{x_u}(x_u)] + E_{x_v \thicksim p_{data}(x_v)}[log (1 - D_{x_u}(G(x_v)))]
$$

这里与一般的GAN不同的是，loss并没有采用nll loss，而是采用了least-square loss（最小二乘法loss），

train for generator to minimize:$E_{x_v \thicksim p_{data}(x_v)}[(D_{x_v}(x_v) - 1)^2]$

train for distinguish to minimize:$E_{x_u \thicksim p_{data}(x_u)}[(D_{x_v}(x_u) - 1)^2] + :E_{x_v \thicksim p_{data}(x_v)}[D_x(x_v)]$

cycle-consistency loss

cycle-consistency loss用来表示某一模态生成指定模态的特征后再由其生成会原本模态的特征，该生成的特征与原有的特征的距离，即由txt生成的img‘，由img’生成txt‘，对比txt与txt’之间的距离，定义如下：
$$
L_{cyc}(G, F)=E_{x_v \sim p_{data}(x_u)}[||F(G(x_u))-x_u||1] +E{x_u \sim p_{data}(x_u)}[||G(F(x_v))-x_u||_1]
$$

deep generative hashing

这里考虑的是由不同模态间的feature map生成hash 与 hash 生成 feature map过程产生的loss

  1. hash 生成 feature map

我们先做如下定义：

$P_u:h_u \to x_v$， denoted as $p(x_v|h_u)$ $P_v:h_v \to x_u$， denoted as $p(x_u|h_v)$

training

本文在训练阶段中，所有经过辨别器D的数据都是之前生成的数据(可靠性高)，不是新生成的数据，这样能够保证网络的稳定性。同时超参数λ=10，学习率从前100epoch的0.0002动态降低至0

conclusion

本文利用了cycle gan并利用在跨模态上，同时结合SGH为hash与模态特征之间提供数学基础，利用cycle gan的特性靠近不同模态间的特征距离以及利用hash与特征的互相生成，靠近hash与特征的距离。

但本文的讲解方面有一点晦涩难懂，并且对于网络模型结构的描述也不够清晰。同时改论文结合了多个网络模型，且模型之间不是端到端的训练，增加了训练的难度。

improve

个人理解，作者认为成对的训练数据是不可靠的，所以通过cycle-gan的方式来生成同类型下其他模态的特征，但缺少了特征提取的loss，导致对某一个模态的输入查询有明显效果，但其他模态的输入查询会由于特征提取的不够精确，导致效果不佳，所以建议不仅仅在feature map之间进行相互生成，同时在输入与feature map之间做到相互生成，或输入与输入之间的相互生成。当然作者也有自己的考虑，根据本文的参考文献31中提及了text 生成 image的模型结构，但作者并没有如此设计，应该是有其理由。

Multi-Level Factorisation Net for Person Re-Identification

2019-04-11

from CVPR 2018

abstract

本文利用了多层结构提取不同在不同卷积深度下的表现最优的特征，然后对特征进行整合用来实现对行人的Re-ID。在Re-ID和物体检测上都表现出了很好的效果，尤其在Re:ID上达到了sota

introduction

对于Re-ID问题，难点在于同一人的姿态、外观等细小特征表现不同，同时对于输入的数据中还包含了摄像头的角度不同等因素，导致Re-ID成为难题。
对于每一个人，可以通过图像进行一些信息的提取，例如下图:

其中很多方法是通过对于这可见的语义进行深层的特征提取，然后通过这些通过高层次的语义进行识别，但这样存在高成本和高错误的问题。
另外的方法是通过注意力机制对特征进行融合，但这样的做法很盲目，无法获取每层能够获取的最有效的因素特征。(例如:浅层的特征一般用于提取衣服颜色，高层的特征用于提取性别)
本文中采用了多级别因素提取网络，网络结构如下：

利用该网络将得到不同层次中可以用来分类的因素，和与分类无关的因素。

network architecture and optimisation
network architecture

MLFN(Multi-Level Factorisation Net)网络由多个block构成，每个block(B)表示一个语义级别；每个block中包含了多个Factor Module(FM)用来提取每个因素的特征，每个FM中包含了独立多个卷积层与pooling层。Factor Selection Module(FSM)因素选择模型，用来选取该block中生效的因素，对于不生效的因素，网络进行冻结处理。网络结构与ResNeXt相似，但不同的是FM之间没有共享权重。

1.FM

对于每个block中，包含了Kn个FM模型，每个block中的FM的个数不同，对于每个FM的输出如下
$$
M_{n,i}\in\R^{H_n×W_n×C_n} ~ M_{n,i}是具有H_n*W_n大小C_n通道的特征
$$

2.FSM

每个block中的FSM用来输出一个FM选择向量
$$
S_n = \delta(\overline{A}_n)whereS_n\in\R^{1×K_n} \
其中\overline{A}_i为第i个FM在经过激活函数之前的值，\delta(·)表示sigmoid函数
$$

3.Block

因此每一层block的输出为{Mn, Sn}，对于输出的详细格式如下:
$$
Y_n = M_n \times 4 S_n + X_n \
其中M_n\in\R^{H_n\times W_n\times C_n\times K_n}, \times4S_n表示第4个维度与S_n进行张量乘法，即dot(dim=4)\
X_n为来自上一层的输出即Y_{n-1}，最底层时X_n为输入的图像\
Y_n \in \R^{H_n\times W_n\times C_n}
$$
每层的block的输出特征都会被下一层所接收，这样所有的因素特征都会集中在最顶层的输出中。

4.Factor Signature

在经过blocks的特征提取后，通过Facotr Signature层将每层的FSM进行融合，融合方式如下：

这里的输出向量大小与输出的特征图大小无关，只与block的个数与block中FM的个数相关

5.Fusion

该结构的特点之一就是不需要修改维度就可以进行融合，因为所有因素的特征都在最顶层block的输出YN中，所以我们只需要将YN与FS的输出结可，这里采用了一种类似全连接的方式，使用函数T将两个维度不同的张量映射到维度为1×d的张量上

Optimisation

对于模型的优化，采用BP为基础，但由于loss通过BP传播到底层时，梯度值的影响降低，所以采用了short connection的方式减缓BP从顶到底的衰弱。
由于FS与FSM的输出相连接，所以每层的FM与FSM相连接来使BP能够通过FSM传到到FM中。
BP求导公式如下：
$$
M_{n,i} = F_{n,i}(x_n;\theta_{n,i})其中\theta_{n,i}为第n层第i个FM的参数\
由于\frac{\partial Y_n}{\partial F_{n,i}} = S_{n,i}
\frac{\partial L}{\partial \theta_{n,i}} = \frac{\partial L}{\partial Y_n}
\frac{\partial Y_n}{\partial F_{n,i}}\frac{\partial F_{n,i}}{\partial \theta_{n,i}}\
得：\frac{\partial L}{\partial \theta_{n,i}} = S_{n,i}\frac{\partial L}{\partial Y_n}
\frac{\partial Y_n}{\partial F_{n,i}}
$$
其中对于无效得因素，其FM得参数会通过Sn,j -> 0来判断是否冻结参数。

对于Re-ID问题，每一个人就是一个分类，所以对于我们得到得最终输出结果R还需要映射到1*n_class的空间内，然后通过L2距离来进行分类

train

在训练的时候，包含了16个blocks，每层的FM个数为32个，最后输出的R的维度d采用1024
对于mini-batch=64时，采用Adam优化器，mini-batch=256时，采用SGD优化器

improvement

本文中在进行对比的时候提到了DPFL模型，其效果与本文结果相近，其中采用了multi-branch来获取不同感受野下的模型，作者认为如果将该部分融入模型模型会有更好的效果。

对于MLFN的输出结果R，可以通过无监督聚类的方式，增强输出向量的关系。

对于本文的损失函数，仅有输出R的L2距离，而对于每个因素的特征训练，可以通过控制其他因素相同，单独训练某一个因素，进而提高每个FM对于因素的分辨能力。

图像处理----图像金字塔

2018-11-21

图像金字塔是一种以多分辨率来解释图像的有效但概念简单的结构。应用于图像分割，机器视觉和图像压缩。一幅图像的金字塔是一系列以金字塔形状排列的分辨率逐步降低，且来源于同一张原始图的图像集合。其通过梯次向下采样获得，直到达到某个终止条件才停止采样。金字塔的底部是待处理图像的高分辨率表示，而顶部是低分辨率的近似。我们将一层一层的图像比喻成金字塔，层级越高，则图像越小，分辨率越低。

  1. 高斯金字塔

    高斯金字塔式在Sift算子中提出来的概念，首先高斯金字塔并不是一个金字塔，而是有很多组（Octave）金字塔构成，并且每组金字塔都包含若干层（Interval）
    高斯金字塔构建过程：

    1. 先将原图像扩大一倍之后作为高斯金字塔的第1组第1层，将第1组第1层图像经高斯卷积（其实就是高斯平滑或称高斯滤波）之后作为第1组金字塔的第2层，高斯卷积函数为：

    ​ 对于参数σ，在Sift算子中取的是固定值1.6

    2. 将σ乘以一个比例系数k,等到一个新的平滑因子σ=k*σ，用它来平滑第1组第2层图像，结果图像作为第3层。
3. 如此这般重复，最后得到L层图像，在同一组中，每一层图像的尺寸都是一样的，只是平滑系数不一样。它们对应的平滑系数分别为：0，σ，kσ，k^2σ,k^3σ……k^(L-2)σ。
4. 将第1组倒数第三层图像作比例因子为2的降采样，得到的图像作为第2组的第1层，然后再对第2组的图像做如上高斯平滑(高斯卷积) ... 如此得到多组图像

    在同一组内，不同层图像的尺寸是一样的，后一层图像的高斯平滑因子σ是前一层图像平滑因子的k倍；在不同组内，后一组第一个图像是前一组倒数第三个图像的二分之一采样，图像大小是前一组的一半；

    高斯金字塔是通过高斯平滑和亚采样获得一些列下采样图像，也就是说第K层高斯金字塔通过平滑、亚采样就可以获得K+1层高斯图像，高斯金字塔包含了一系列低通滤波器，其截至频率从上一层到下一层是以因子2逐渐增加，所以高斯金字塔可以跨越很大的频率范围。

    对图像的向下取样(缩小图像, 分辨率提高)
    为了获取层级为 G_i+1 的金字塔图像，我们采用如下方法:

    ​ <1> 对图像G_i进行高斯内核卷积

    ​ <2> 将所有偶数行和列去除
    ​ ==结果图像只有原图的四分之一==, 向下取样会逐渐丢失图像的信息

    对图像的向上取样(放大图像, 分辨率降低)
    ​ <1> 将图像在每个方向扩大为原来的两倍，新增的行和列以0填充

    ​ <2> 使用先前同样的内核(乘以4)与放大后的图像卷积，获得 “新增像素”的近似值

  2. DOG金字塔(差分(Difference of Gaussian)金字塔)
    DOG金字塔的第1组第1层是由高斯金字塔的第1组第2层减第1组第1层得到的。以此类推，逐组逐层生成每一个差分图像，所有差分图像构成差分金字塔。概括为DOG金字塔的第o组第l层图像是有高斯金字塔的第o组第l+1层减第o组第l层得到的。

    然后对这些DOG图像进行==归一化==，可有很明显的看到差分图像所蕴含的特征，并且有一些特征是在不同模糊程度、不同尺度下都存在的，这些特征正是Sift所要提取的“稳定”特征

  3. 拉普拉斯金字塔（应用于图像融合）(减少信息丢失)
    是高斯金字塔的修正版，为了还原到原图。通过计算残差图来达到还原。

    拉普拉斯金字塔是通过源图像减去==先缩小后再放大==的图像的一系列图像构成的(缩小放大过程如上)
