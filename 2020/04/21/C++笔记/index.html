运行环境

程序编译

  1. 编译流程

    1. 编译预处理：替换源代码中的宏，生成预编译文件(.i)
    2. 优化：根据语法分析，对代码中例如循环、公共表达式等进行优化重排并生成对应的汇编代码(.s)
    3. 汇编：将汇编代码生成对应的机器指令，并生成目标代码(.o)

  2. 连接流程

    将外部调用连接，使其统一合并为一个执行体(可执行文件)，分为以下两种链接。

    • 静态链接：代码从静态链接库中拷贝在最终执行文件中
    • 动态链接：代码放到动态链接库(dll)或共享对象的某个目标文件中，连接程序记录对象名字和dll登记信息，执行时dll中的内容全部映射到执行进程的虚地址空间。

语法特性

类型转换

  • 隐式转换：系统自动进行

  • 显示转换：强制类型转换

  • C++命名类型转换：

    1. static_cast: 静态转换， 编译时转换(检测)，保证安全性，用于非多态类型转换

      • 基础类型转换：int to float, etc
      • 子类向基类转换
      • 无类型指针与其他类型指针互相转换：void* to int*, int * to void *, etc

    2. const_cast: 去掉指针或引用的const和volatile属性，不要修改转换的对象(行为未定义，结果未知)

    3. reinterpret_cast: 编译时类型转换，重新解读，处理无关类型，不进行检查(不报错)

      • 将int转化为指针或将一种指针转化为另外一种指针
      • 指针转换为int, long, long long类型

    4. dynamic_cast: 运行时转换(检查)，作用于子类和父类之间的指针和引用转换，只能作用于有虚函数的类

      • 转换类型为指针：失败返回NULL
      • 转换类型为引用：失败抛出异常

左值和右值

  • 左值：变量和引用本身，出了常量左值引用外不能引用右值

attention： ++i 是左值 i++ 是右值(i++中间生成临时变量)

折叠表达式(C++17)

在C++17中提出的一个二元运算符来对形参包进行规约

  • 形参包(B) op … 一元右折叠： $(B_1op(B_2op(…op(B_{n-1}op~B_n)))$
  • … op 形参包(B) 一元左折叠: $(((B_1opB_2)op…)opB_{n-1})opB_n)))$
  • 形参包(B) op … op 初值(I) 二元右折叠: $(B_1op(B_2op(…op(B_{n-1}op(B_nopI))))$
  • 初值(I) op … op 形参包(B) 二元左折叠: $((((IopB_1)opB_2)op…)opB_{n-1})op~B_n)))$

一元折叠在用于空包展开时，允许下列运算符：

  • &&逻辑与 空包值 true
  • || 逻辑或 空包值 false
  • ，逗号运算符 空包值 void()

内存管理

new delete

C++中通过new动态分配空间，delete释放空间，对应C中的malloc和free。

对于系统自动分配的对象(非new创建)，对象存放在栈(空间小)中，在生命周期结束后自动回收；
对于动态分配的对象(new创建)，对象存放在堆(大小约=内存大小)中，需要手动释放或程序结束时回收。

new/delete是申请/释放单个对象的空间，new在申请失败时抛出异常；malloc/free是申请/释放连续的空间，malloc在申请失败时返回NULL指针；new[]/delete[]是申请/释放连续空间。

内置对象(int, float,…)通过new[]分配内存，通过delete删除指针；同样，如果自定义类没有自定义析构函数，则可以和内置类型一样通过delete释放。定义析构函数的类在new申请空间时，会额外定义4字节(int)记录数组元素个数；相反对于内置类和未定义析构函数的类new申请时不会记录数组元素个数

内存池

封装内存分配，减少内存碎片，提高内存使用效率。

思路：利用链表，一次分配多个对象大小，每个对象块指向下一个块，new(malloc)申请的内存是连续的，如果一次申请的空间不够，需要再次申请则新的地址和已申请的地址不连续，通过链表将不连续的地址连接。

实施：重载new和delete运算符

优化：

智能指针

直接接收new创建内容的指针称为裸指针，其需要在使用周期中全程维护，手动释放。
智能指针：在裸指针基础进行包装(类模板)，能够自动释放

  • auto_ptr(C++98) 弃用
  • unique_ptr(C++11)：独占指针，同一时间内只能一个指针指向某个对象
  • shared_ptr: 共享指针，多个指针指向同一对象，当最后的指针被销毁时，对象才被释放(引用计数)
    • weak_ptr: 用来被shared_ptr管理对象存在非拥有性引用，访问对象前需转换为shared_ptr

智能指针的思想就是通过栈中的对象来控制堆中的对象，在栈中申请的对象会在生命周期结束时自动释放，而堆中的对象需要手动释放，这时将堆中的对象在栈中创建一个映射，使得利用栈的自动释放来管理堆。

std::thread()

thread通过传入一个可执行的对象来创建并允许线程，传入的执行对象的参数最好为右值。

joinable

检查线程是否可被join。如果是一个活动的执行进程时可以被join的，默认构造函数创建的thread是不可以被join的。如果一个线程执行完但没有被join，其依然是可以被join的。

detach

一个子线程被detach后，子线程的控制器移交给C++后台守护进程，父线程不再管理，子线程执行完后自动释放资源。若该线程的joinable=false,调用该方法会返回std::system_error异常。

  • 当线程被detach后，joinable=false，detach后的线程不能调用join。

  • 在detach子线程时，不建议使用指针作为执行对象的参数(main中修改后detach线程也会被修改)

  • 在detach子线程时，传入参数为引用时也会被改为值传递(main和子线程数据不共享)

数据有效性

原子性

数据在操作时不能被其他任务打断，完整的完成某一个或一系列操作。

gcc提供了一些列原子对象和相关的原子操作(针对于变量的操作不是代码段)

std::atomic

对于多个线程操作通过一个对象时，由于对象的操作不是原子性，导致线程可能获取其他线程处理过程中对象的中间状态。

如果需要对代码段需要原子性保证，则需要通过信号量和锁的方式保证。

有序性

在gcc编译时会对代码的执行顺序或内容进行优化，可能导致前面的的代码在后面的代码之后执行。

gcc实现了volatile关键字，用来声明一个对象类型是”易变”的，在编译时volatile修饰的对象相关指令不会被重排,保证只在内存中读写；volatile适合于信号处理函数之间的交流，但不适合线程交流。

操作原子性

信号量

在std::mutex中定义了信号量，通过对信号量进行上锁或释放锁的方式实现操作的原子性。

自动释放锁

在标准库中实现了能够在对象释放后自动释放的锁(通过栈的对象来管理堆中的锁)

被lock_guard维护的信号量不能再被lock或unlock

多信号量锁

在标准库中定义了std::lock()函数能够同时锁多个信号量(≥2)，当其中任何一个互斥量上锁失败时，会把所有已上锁的信号量释放(防止死锁)

如果一个信号量已经被lock锁上，若需要lock_guard自动释放对象，则可以在创建guard时添加标识(结构体)

标识有如下几种：

  • std::adopt_guard：标识mutex已被lock
  • std::try_to_lock: 标识mutex尝试加锁，如果失败则直接返回，不堵塞线程
  • std::defer_lock: 标识mutex不能提前加锁
unique_lock

功能更多的lock_guard，但是会消耗更多内存。功能接近unique_ptr自能指针，可以动态更改管理的信号量，修改信号量的所有权

std::conditional_variable

该类是一个同步原语，用于堵塞一个或同时堵塞多个线程，直到其他线程修改共享变量(条件)并通知conditional_variable对象。一般配合独立锁信号量实现(unique_lock)。（不能复制复制operator= delete）

后台任务

通过std::async启动一个异步任务，并返回一个std::future对象，当异步任务执行完毕后可以从future中获取返回结果。

std::future

future对象用来和一个任务绑定，用来接收任务的返回结果。并提供了如下方法：

  • get() 堵塞获取结果，如果任务没完成则堵塞当前调用线程
  • wait() 堵塞当前线程直到任务完成
  • valid() 判断future是否为共享状态(多个future不能指向同一对象，但shared_future可以)
  • share() 转移当前的监控到一个shared_future上，然后自身不再监控，返回一个shared_future
shared_future

允许多个std::shared_future指向同一个任务，并且可以复制。

shared_future和future类似于shared_ptr和weak_ptr之间的关系，通过引用计数的方式来记录引用关系。

std::async

通过async调用一个可执行对象来创建一个异步任务，并返回一个与之绑定的future对象。一般创建后会直接调用可执行对象开始任务，也可以控制任务类型。

异步任务不代表一定会创建一个新线程，当资源紧张时异步任务也不会创建新线程

std::packaged_task

将一个任务进行打包，变成一个对象，方便thread或async调用

线程间数据共享

  • 通过全局变量实现join线程之间的数据共享(引用)

  • 通过在放置在堆中的对象数据，实现detach线程数据共享(指针传递，需要手动释放)

    • 智能指针作为参数时，如果指针所在的线程结束，栈回收时会释放智能指针指向堆的地址释放
std::promise

  1. self-contained 头文件：头文件应该满足作为一个单独头文件，不依赖其他头文件

  3. 将所有的函数置于命名空间中，不要创建裸的全局函数

  4. 函数变量尽量置于最小作用域内，并在声明时初始化

  5. 禁止定义静态存储周期非POD(Plain Old Data: int,char,float及其指针)变量,禁止定义使用含有副作用的函数初始化POD全局变量：多编译单元的静态变量执行时构造和析构顺序顺序未明确

类

  1. 构造函数中不要调用虚函数，也不要在无法报错时进行可能失败的初始化。(必须调用虚函数时用工厂或Init)
  2. 不要定义隐式类型转换，对于转换运算符和单个参数的构造函数使用explicit关键字
  3. 如果不需要拷贝/移动构造函数，则应该禁用
  4. 只有数据成员的时候使用struct，其他一律用class
  5. 组合比继承更合理，继承一般用于：子类继承父类的方法实现、子类继承父类的方法名称
  6. 接口一般以Interface为后缀，只有纯虚函数和静态函数，没有非静态数据成员，没有定义任何构造函数(如果有没不带参数且为protected)
  7. 多重继续只允许在最多有一个非抽象基类，且其他基类都是接口(纯接口类)
  8. 不建议重载运算符，也不要定义字面量
  9. 所有数据成员声明为private，除非是static const类型成员
  10. 将public部分放在最前，然后是protected，最后是private，每次缩进1个空格

函数

  1. 输入参数在前，输出参数在后(Google Code硬规定：输入参数为值参或cost引用，输出参数为指针)
  2. 函数长度不宜过长，40行为上限
  3. 所有按引用传递的参数必须加上const
  4. 缺省函数只允许在非虚函数中使用，并且要保证缺省函数的值始终一致
  5. 只有在常规写法不便的时候才使用后置语法(auto foo(int x) -> int;)

格式

  1. 每行字符数不超过80

  2. 非ASCII字符必须用UTF-8编码

  3. 使用空格，每次俩个空格，不适用tab

  4. 函数的放回类型和函数名在同一行，参数也尽量在同一行，放不下就对形参分行

    函数名和(之间没有空格；()与参数之间没有空格，{在最后一个参数的末尾处，不另起新行，缺省缩进为2空格

  5. lambda表达式对形参和函数体的格式和其他函数一致，捕获列表用,隔开，引用捕获变量名和&之间不留空格

  6. 函数调用尽量在一行内调用，如果参数放在次行则缩进4空格

  7. 条件语句在()内不使用空格，else写在if的}同一行

  8. switch语句中可以用{}分段来加强case的表达

  9. 在只有一句的循环中可以不用{}，空循环体必须用{}或contine

  10. 指针*/地址操作符&之后不能加空格，成员符号.和指针成员符->后也不能加空格

  11. 布尔表达式如果超过行宽，则&&操作符放在行尾

  12. 函数返回时，return后面的返回不要加()

  13. 预处理指令不要缩进，从行首开始

  14. 命名空间内部不要有额外缩进

注释

  1. 文件注释： 文件开头添加版权公告，描述文件的内容，如果内部实现的内容已有注释就不用在文件添加

  2. 类注释：描述功能和用法

  3. 函数注释：函数声明处描述函数功能，定义处描述函数实现

    声明处描述：输入输出；类成员函数是否需要保持引用参数，是否释放参数；函数是否分配必须由人为释放的空间；参数是否可为空指针；函数是否有隐患；函数是否可重入，同步前提是什么；

  4. 变量注释：一般根据变量名说明含义，类数据成员和全局变量都应该有注释和说明用途

  5. 实现注释：代码中晦涩的地方加注释

  6. TODO注释：对临时或已经够好的代码使用TODO注释,（）内写上注释人相关信息

  7. 弃用注释：DEPRECATED 标记接口弃用，同TODO一样，()内写上注释人相关信息

其他

  1. 动态分配的对象最好有单一且固定的所有主，并通过智能指针传递所有权(std::unique_ptr)
  2. 所有按引用传递的参数必须加上const：明确不接受NULL指针
  3. 右值引用只在移动构造函数和赋值操作时使用，不要使用std::forward
  4. 可以定义个const string&参数的函数，然后用const char*参数去重载。尽量在函数名体现参数信息避免重载
  5. 不允许使用缺省函数
  6. 不允许使用变长数组和alloca()：容易引起内存越界bug
  7. 不使用C++异常
  8. 禁用RTTI
  9. 使用static_cast作为明确的类型转换，不要使用(int)或int()
  10. 只在日记中使用流：用流打印对象不需要关心类型
  11. 迭代器使用++i前缀形式自增，–i自减运算符
  12. 能用const和constexpr最好使用，constexpr用来定义真正的常量并初始化
  13. 整型在C++中没有明确定义，默认为short 16bit, int 32bit, long 32bit, long long 64bit
  14. 宏尽量用内联函数、枚举和常量替换
  15. int 0, float 0.0指针用nullptr或NULL, 字符串用’\0’
  16. 用sizeof(varname)代替sizeof(type)
  17. auto只用在局部变量
  18. 可以用列表初始化，自定义类型可接受std::initializer_list<T>,不要用初始化列表初始auto变量
  19. 不要默认使用lambda捕获，所有捕获显示写出来
  20. boost库中部分允许使用
